## 🔧 第三次修复：CGEventTap 拦截

### 根本原因

**rdev** 只能**被动监听**事件，无法阻止事件传播：

```
用户按 ` 键
  ↓
rdev 监听到 ✅
  ↓
事件继续传播到系统 ❌
  ↓
系统输入 ` 字符 ❌
```

### 解决方案

使用 **CGEventTap** 创建**拦截型**事件监听器：

```
用户按 ` 键
  ↓
CGEventTap 拦截 (系统级别)
  ↓
检测到 keycode 50 (` 键)
  ↓
触发 Aleph 回调 ✅
  ↓
返回 None (吞噬事件) ✅
  ↓
系统收不到事件 → 不输入字符 ✅
```

## 📝 技术细节

### CGEventTap 参数

```rust
CGEventTap::new(
    CGEventTapLocation::HID,                    // 硬件事件流（系统级别）
    CGEventTapPlacement::HeadInsertEventTap,    // 插入到事件队列头部
    CGEventTapOptions::Default,                 // 拦截模式（可修改/删除事件）
    vec![CGEventType::KeyDown],                 // 监听按键按下事件
    event_callback,                             // 回调函数
)
```

### 拦截机制

```rust
match event_type {
    CGEventType::KeyDown => {
        let keycode = event.get_integer_value_field(EventField::KEYBOARD_EVENT_KEYCODE);

        if keycode == 50 {  // macOS keycode for `
            trigger_aether_callback();
            return None;  // 🔑 吞噬事件，阻止传播
        }

        Some(event.to_owned())  // 其他键放行
    }
    _ => Some(event.to_owned())
}
```

### macOS 键码参考

| 键 | Keycode |
|----|---------|
| ` (grave/backtick) | 50 |
| 1 | 18 |
| Escape | 53 |
| Delete | 51 |

## 📋 如果测试失败

### 现象 1：` 键仍然输入字符
**可能原因**：
- CGEventTap 创建失败（权限问题）
- EventTap 未正确拦截事件

**调试**：
1. 查看 Console.app 是否有 "[EventTapListener] ERROR"
2. 确认 Accessibility 权限已授予
3. 检查 CGEventTap 是否成功创建

### 现象 2：` 键不输入，但 Aleph 未触发
**可能原因**：
- 回调函数未正确执行
- Keycode 检测错误

**调试**：
1. 查看 Console.app 是否有 "Detected ` key press"
2. 如果没有，可能 keycode != 50

### 现象 3：应用崩溃
**可能原因**：
- CGEventTap 回调中的线程安全问题
- Run loop 问题

**调试**：
1. 提供崩溃日志 (.ips 文件)
2. 查看崩溃堆栈
