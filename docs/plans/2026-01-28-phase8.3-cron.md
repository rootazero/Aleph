### Features Implemented

- **Standard cron expressions** - 5-field format (min hour day month weekday)
- **SQLite persistence** - Jobs and run history stored in `~/.aleph/cron.db`
- **Concurrent execution limits** - Configurable max parallel jobs
- **Job lifecycle management** - Add, update, delete, enable, disable
- **Run history** - Track job execution results with timestamps
- **Timeout handling** - Configurable job execution timeout
- **History cleanup** - Automatic cleanup of old run records

### Configuration

```toml
[cron]
enabled = true
db_path = "~/.aleph/cron.db"
check_interval_secs = 60
max_concurrent_jobs = 5
job_timeout_secs = 300
history_retention_days = 30
```

### Cron Expression Format

```text
┌───────────── minute (0 - 59)
│ ┌───────────── hour (0 - 23)
│ │ ┌───────────── day of month (1 - 31)
│ │ │ ┌───────────── month (1 - 12)
│ │ │ │ ┌───────────── day of week (0 - 6, Sunday = 0)
│ │ │ │ │
* * * * *
```

**Examples:**
- `0 9 * * *` - Every day at 9:00 AM
- `*/15 * * * *` - Every 15 minutes
- `0 0 * * 0` - Every Sunday at midnight
- `0 9 1 * *` - First day of every month at 9:00 AM

## API

### CronJob

```rust
pub struct CronJob {
    pub id: String,
    pub name: String,
    pub schedule: String,    // Cron expression
    pub agent_id: String,    // Target agent
```

### JobStatus

```rust
pub enum JobStatus {
    Pending,    // Job never run
    Running,    // Currently executing
    Success,    // Completed successfully
    Failed,     // Execution failed
    Skipped,    // Skipped (previous run still in progress)
    Timeout,    // Execution timed out
}
```

## Architecture

```
┌─────────────────────────────────────┐
│           CronService               │
├─────────────────────────────────────┤
│  ┌────────────┐  ┌───────────────┐ │
│  │  Scheduler │  │  Job Executor │ │
│  │  (ticker)  │  │  (callback)   │ │
│  └──────┬─────┘  └───────┬───────┘ │
│         │                │         │
│         ▼                ▼         │
│  ┌────────────────────────────────┐│
│  │      SQLite Database           ││
│  │  • cron_jobs (definitions)     ││
│  │  • cron_runs (history)         ││
│  └────────────────────────────────┘│
└─────────────────────────────────────┘
```

### Execution Flow

1. Scheduler checks for due jobs every `check_interval_secs`
2. For each enabled job with matching schedule:
   - Check if already running (skip if so)
   - Acquire semaphore permit (defer if at max concurrent)
   - Create JobRun record with status=Running
   - Execute callback with timeout
   - Update JobRun with result (Success/Failed/Timeout)
3. Cleanup old history based on `history_retention_days`

## Usage

### Starting with Gateway

```bash
# Build with cron feature
cargo build --features cron

# Run gateway (cron automatically starts if configured)
cargo run --features cron --bin aleph-gateway
```

### Programmatic Usage

```rust
use alephcore::cron::{CronService, CronConfig, CronJob};

// Create service
let config = CronConfig::default();
let mut service = CronService::new(config)?;

// Set executor callback
service.set_executor(Arc::new(|job_id, agent_id, prompt| {
    Box::pin(async move {
        // Execute agent with prompt
        Ok("Job completed".to_string())
    })
}));

// Add a job
let job = CronJob::new(
    "Daily Report",
    "0 18 * * *",
    "main",
    "Generate end-of-day summary"
);
service.add_job(job).await?;

// Start scheduler
service.start().await?;
```

## Database Schema

### cron_jobs

| Column | Type | Description |
|--------|------|-------------|
| id | TEXT PRIMARY KEY | Unique job ID |
| name | TEXT | Human-readable name |
| schedule | TEXT | Cron expression |
| agent_id | TEXT | Target agent ID |
| prompt | TEXT | Message to send |
| enabled | INTEGER | 1=enabled, 0=disabled |
| timezone | TEXT | Optional timezone |
| tags | TEXT | JSON array of tags |
| created_at | INTEGER | Unix timestamp |
| updated_at | INTEGER | Unix timestamp |

### cron_runs

| Column | Type | Description |
|--------|------|-------------|
| id | TEXT PRIMARY KEY | Unique run ID |
| job_id | TEXT | Foreign key to cron_jobs |
| status | TEXT | pending/running/success/failed/skipped/timeout |
| started_at | INTEGER | Unix timestamp |
| ended_at | INTEGER | Unix timestamp (0 if running) |
| duration_ms | INTEGER | Execution duration |
| error | TEXT | Error message if failed |
| response | TEXT | Truncated response |

## Limitations

1. **No distributed locking** - Single instance only
2. **No timezone support** - Uses local time (UTC planned)
3. **No retry logic** - Failed jobs not automatically retried
