# P4.3 Core Plugins Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Implement 6 core plugins that showcase Aleph's plugin ecosystem and provide real utility — voice calling, diagnostics, memory analytics, batch LLM, code diff, and phone control.

**Architecture:** Each plugin is a standalone package in `plugins/` using the plugin infrastructure built in P4 (manifest, JSON-RPC IPC, @aleph/plugin-sdk). Node.js plugins use stdio JSON-RPC; WASM plugins use Extism PDK. Plugins declare tools/hooks/services in `aleph.plugin.toml` and implement handlers. All plugins are independent — no cross-plugin dependencies.

**Tech Stack:** Node.js (4 plugins), Rust/WASM via Extism PDK (2 plugins), JSON-RPC 2.0, OpenTelemetry JS SDK, WebRTC/SIP stubs, ADB CLI, `similar` crate for diffs.

**Reference:**
- Example plugin: `examples/plugins/media-video/`
- Plugin SDK types: `packages/plugin-sdk/src/types.ts`
- Manifest format: `core/src/extension/manifest.rs`
- Node.js runtime: `core/src/extension/runtime/node_runtime.rs`

System health monitoring and performance diagnostics. Exposes tools for checking Aleph's runtime state, collecting OpenTelemetry-compatible traces, and reporting performance metrics.

**Files:**

- Create: `plugins/diagnostics/README.md`

**Step 1: Create manifest**

```toml
[plugin]
id = "diagnostics"
name = "Aleph Diagnostics"
version = "0.1.0"
description = "System health monitoring, performance diagnostics, and OpenTelemetry trace collection"
kind = "nodejs"
entry = "src/index.js"

[permissions]
shell = true
filesystem = "read"
network = true

[[tools]]
name = "system_health"
description = "Check system health: CPU, memory, disk, uptime, and Aleph process stats"
handler = "systemHealth"

[tools.parameters]
type = "object"
properties = {}

[[tools]]
name = "perf_profile"
description = "Profile performance of the last N operations with timing breakdown"
handler = "perfProfile"

[tools.parameters]
type = "object"
properties.last_n = { type = "number", description = "Number of recent operations to analyze", default = 10 }
properties.format = { type = "string", description = "Output format: summary, detailed, or json", default = "summary" }

[[tools]]
name = "trace_export"
description = "Export collected traces in OpenTelemetry-compatible JSON format"
handler = "traceExport"

[tools.parameters]
type = "object"
properties.duration_secs = { type = "number", description = "Export traces from the last N seconds", default = 300 }
properties.service_filter = { type = "string", description = "Filter by service name (regex)" }

[[tools]]
name = "log_analyze"
description = "Analyze recent log patterns, errors, and warnings"
handler = "logAnalyze"

[tools.parameters]
type = "object"
properties.level = { type = "string", description = "Minimum log level: debug, info, warn, error", default = "warn" }
properties.last_n = { type = "number", description = "Number of log entries to analyze", default = 100 }
properties.pattern = { type = "string", description = "Regex pattern to filter logs" }

[[hooks]]
event = "SessionStart"
kind = "observer"
handler = "onSessionStart"
priority = "low"

[[hooks]]
event = "SessionEnd"
kind = "observer"
handler = "onSessionEnd"
priority = "low"

[[services]]
name = "metrics-collector"
description = "Background metrics collection service"
start_handler = "startCollector"
stop_handler = "stopCollector"
```

**Step 2: Create package.json**

```json
{
  "name": "@aleph/diagnostics",
  "version": "0.1.0",
  "description": "System diagnostics and OpenTelemetry trace collection for Aleph",
  "main": "src/index.js",
  "scripts": {
    "test": "node src/__tests__/test.js"
  }
}
```

**Step 3: Implement system collector** (`src/collectors/system.js`)

Collects CPU load, memory usage, disk space, uptime via Node.js `os` module and child_process for platform-specific commands.

**Step 4: Implement process collector** (`src/collectors/process.js`)

Collects Aleph server process info: PID, RSS, heap, event loop lag, open file descriptors.

**Step 5: Implement formatters** (`src/formatters.js`)

Format metrics as human-readable summary, detailed table, or OpenTelemetry-compatible JSON spans.

**Step 6: Implement main entry** (`src/index.js`)

JSON-RPC stdio listener with handler map. Session hooks track session start/end times. Metrics collector service stores time-series data in memory (ring buffer, last 1 hour).

**Step 7: Test and validate**

Run: `cd plugins/diagnostics && node src/index.js` (manual stdin test)
Run: `cargo run --bin aleph-cli -- plugin validate plugins/diagnostics`

