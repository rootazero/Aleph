## Implementation Plan

### Phase 8.4a: Core Browser Service

1. **Dependencies** (`Cargo.toml`)
   - `chromiumoxide` - Rust CDP client
   - Feature flag: `browser`

2. **Configuration** (`browser/config.rs`)
   - BrowserConfig: executable path, CDP port, headless, timeout
   - BrowserProfile: multi-profile support (future)

3. **Service** (`browser/mod.rs`)
   - BrowserService: launch, connect, shutdown
   - Page management: tabs, navigation, screenshots

### Phase 8.4b: Browser Actions

4. **Actions** (`browser/actions.rs`)
   - navigate(url) - Go to URL
   - click(selector) - Click element
   - type(selector, text) - Type text
   - screenshot() - Capture viewport
   - evaluate(js) - Run JavaScript
   - snapshot() - Get page accessibility tree

5. **Snapshot** (`browser/snapshot.rs`)
   - ARIA tree extraction
   - Element reference generation (e1, e2, ...)
   - Truncation for large pages

### Phase 8.4c: Gateway Integration

6. **Handlers** (`gateway/handlers/browser.rs`)
   - browser.start - Launch browser
   - browser.stop - Terminate browser
   - browser.navigate - Navigate to URL
   - browser.click - Click element
   - browser.type - Type text
   - browser.screenshot - Capture screenshot
   - browser.snapshot - Get page snapshot
   - browser.evaluate - Run JavaScript
   - browser.tabs - List/manage tabs

## API Design

### browser.start

```json
// Request
{"jsonrpc": "2.0", "method": "browser.start", "params": {"headless": true}, "id": 1}

// Response
{"jsonrpc": "2.0", "result": {"ok": true, "cdp_port": 9222}, "id": 1}
```

### browser.navigate

```json
// Request
{"jsonrpc": "2.0", "method": "browser.navigate", "params": {"url": "https://example.com"}, "id": 2}

// Response
{"jsonrpc": "2.0", "result": {"ok": true, "url": "https://example.com", "title": "Example Domain"}, "id": 2}
```

### browser.snapshot

```json
// Request
{"jsonrpc": "2.0", "method": "browser.snapshot", "params": {"format": "aria"}, "id": 3}

// Response
{
  "jsonrpc": "2.0",
  "result": {
    "ok": true,
    "url": "https://example.com",
    "title": "Example Domain",
    "snapshot": [
      {"ref": "e1", "role": "link", "name": "More information..."},
      {"ref": "e2", "role": "heading", "name": "Example Domain", "level": 1}
    ]
  },
  "id": 3
}
```

### browser.click

```json
// Request
{"jsonrpc": "2.0", "method": "browser.click", "params": {"ref": "e1"}, "id": 4}

// Response
{"jsonrpc": "2.0", "result": {"ok": true}, "id": 4}
```

### browser.screenshot

```json
// Request
{"jsonrpc": "2.0", "method": "browser.screenshot", "params": {"full_page": false}, "id": 5}

// Response
{
  "jsonrpc": "2.0",
  "result": {
    "ok": true,
    "image": "base64...",
    "format": "png",
    "width": 1280,
    "height": 720
  },
  "id": 5
}
```

## Key Implementation Details

### Chrome Discovery

1. Check `CHROME_PATH` environment variable
2. Check common paths:
   - macOS: `/Applications/Google Chrome.app/Contents/MacOS/Google Chrome`
   - Linux: `/usr/bin/google-chrome`, `/usr/bin/chromium-browser`
   - Windows: `C:\Program Files\Google\Chrome\Application\chrome.exe`
3. Fall back to `chromium` in PATH

### CDP Connection

```rust
// Launch with CDP
let browser = Browser::launch(
    BrowserConfig::builder()
        .chrome_executable(chrome_path)
        .arg("--remote-debugging-port=9222")
        .arg("--headless=new")
        .build()?
).await?;

// Or connect to existing
let browser = Browser::connect("ws://127.0.0.1:9222").await?;
```

### Element Reference System

Based on Moltbot's approach:
- Generate refs from accessibility tree: `e1`, `e2`, ...
- Cache refs per page for action targeting
- Support CSS selectors as fallback
