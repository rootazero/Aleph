## Usage

### Starting the Gateway

```bash
# With Discord only
cargo run --features discord --bin aleph-gateway

# With all channels
cargo run --features all-channels --bin aleph-gateway
```

### Setting Up a Discord Bot

1. Go to [Discord Developer Portal](https://discord.com/developers/applications)
2. Create a new application
3. Go to "Bot" section
4. Click "Reset Token" to get bot token
5. Enable "Message Content Intent" under "Privileged Gateway Intents"
6. Go to "OAuth2" > "URL Generator"
7. Select scopes: `bot`, `applications.commands`
8. Select permissions: `Send Messages`, `Read Messages/View Channels`
9. Use generated URL to invite bot to your server

### Bot Interaction

The bot responds when:
- Mentioned (@BotName message)
- Message starts with prefix (e.g., `!message`)
- Direct message (if DMs enabled)

## Architecture

```
Discord Gateway
       │
       ▼
┌──────────────────┐
│  DiscordChannel  │
│   (serenity)     │
├──────────────────┤
│ • Event Handler  │
│ • HTTP Client    │
│ • Message Parser │
└────────┬─────────┘
         │
         ▼
  Channel Registry
         │
         ▼
   Gateway Router
```

### Event Flow

1. Discord Gateway sends MESSAGE_CREATE event
2. serenity Handler receives event
3. Handler checks permissions (guild/channel/DM allowlists)
4. Handler extracts text, attachments, metadata
5. InboundMessage sent to channel's mpsc queue
6. Gateway Router processes message through Agent

### Sending Messages

1. Gateway calls `channel.send(OutboundMessage)`
2. DiscordChannel parses conversation_id
3. For DMs: Creates DM channel via API
4. For guilds: Uses channel_id directly
5. serenity HTTP client sends message
