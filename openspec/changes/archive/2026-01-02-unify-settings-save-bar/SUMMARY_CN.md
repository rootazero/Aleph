## 🎯 为什么需要这个变更？

当前Aether的设置界面存在以下问题：

1. ❌ **问题1**：设置修改后立即生效 → 容易误操作
2. ❌ **问题2**：无法预览修改效果就提交 → 用户缺乏信心
3. ❌ **问题3**：测试连接按钮藏在表单底部 → 测试效率低
4. ❌ **问题4**：各设置标签页行为不一致 → 用户体验差

### 用户价值

✅ **明确控制**：用户可以安全地尝试设置，如果不满意可以取消
✅ **快速测试**：直接从侧边栏测试供应商，无需滚动
✅ **防止数据丢失**：导航保护防止意外丢失未保存的修改
✅ **一致体验**：所有设置标签页遵循相同的保存/取消模式

## 🔧 这个变更实现了什么？

### 1. 统一保存栏（所有设置标签页）

```
┌───────────────────────────────────────────────────────────┐
│                设置内容区域                                │
│  （可滚动的表单字段）                                      │
│                                                           │
└───────────────────────────────────────────────────────────┘
┌───────────────────────────────────────────────────────────┐
│  ⚠️ 未保存的修改                  [取消]  [保存]           │  ← 固定底栏
└───────────────────────────────────────────────────────────┘
```

**功能特性**：
- 固定在内容区域底部
- 保存按钮：有修改时显示蓝色，无修改时显示灰色
- 取消按钮：撤销所有修改，恢复到上次保存的状态
- 状态消息：显示"未保存的修改"或保存错误
- 键盘快捷键：Cmd+S（保存）、Escape（取消）

**适用范围**：通用、供应商、路由、快捷键、行为、内存

---

### 2. 每个供应商的测试按钮（侧边栏卡片）

**之前**（旧设计）：
```
┌─────────────────────────────────┐
│ 🔵 OpenAI         [开关]        │
│    OpenAI API                   │
└─────────────────────────────────┘
```

**之后**（新设计）：
```
┌─────────────────────────────────┐
│ 🔵 OpenAI     [📡] [开关]       │  ← 新增测试图标按钮
│    OpenAI API                   │
│    ✓ 连接成功                   │  ← 内联结果显示
└─────────────────────────────────┘
```

**功能特性**：
- 仅图标按钮（SF Symbol: `network`）
- 位于激活开关左侧
- 使用**未保存的表单值**测试（工作副本，非已保存配置）
- 在卡片下方显示内联结果（✓ 绿色或 ❌ 红色）
- 5秒后自动隐藏或在编辑表单时立即隐藏

---

### 3. 导航保护（防止数据丢失）

**警告对话框**（在有未保存修改时切换标签页/关闭窗口时显示）：

```
┌─────────────────────────────────────┐
│  ⚠️  未保存的修改                   │
│                                     │
│  你有未保存的修改。是否在离开前保存？│
│                                     │
│     [保存]  [不保存]  [取消]        │
└─────────────────────────────────────┘
```

**行为**：
- **保存**：提交修改并继续
- **不保存**：放弃修改并继续
- **取消**：留在当前视图

## 📊 技术规格

### 新增组件

1. **UnifiedSaveBar** (`Aether/Sources/Components/Molecules/UnifiedSaveBar.swift`)
   - 所有设置标签页的可复用组件
   - 属性：`hasUnsavedChanges`, `isSaving`, `statusMessage`, `onSave`, `onCancel`

2. **FormStateful 协议** (`Aether/Sources/Utils/FormState.swift`)
   - 定义工作副本 vs. 已保存状态的契约
   - 方法：`save()`, `cancel()`, `loadSavedState()`
   - 计算属性：`hasUnsavedChanges`, `isFormValid()`

3. **NavigationGuard** (`Aether/Sources/Utils/NavigationGuard.swift`)
   - 未保存修改的警告对话框
   - 返回：`.save`, `.discard`, 或 `.cancel`

### 修改的组件

- **ProvidersView**：添加保存栏，在卡片上添加测试按钮
- **GeneralSettingsView**：添加保存栏
- **RoutingView**：添加保存栏
- **ShortcutsView**：添加保存栏
- **BehaviorSettingsView**：添加保存栏
- **MemoryView**：添加保存栏
- **SimpleProviderCard**：添加带内联结果显示的测试按钮

## 🔄 状态管理架构

```
┌─────────────────────────────────────────────────────────┐
│                     设置标签页                          │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  ┌──────────────┐          ┌──────────────┐            │
│  │ 工作副本     │ ←编辑──  │  表单字段    │            │
│  │   (State)    │          │  (UI输入)    │            │
│  └───────┬──────┘          └──────────────┘            │
│          │                                              │
│          │ 比较                                         │
│          ↓                                              │
│  ┌──────────────┐                                      │
│  │ 已保存状态   │                                      │
│  │   (配置)     │                                      │
│  └───────┬──────┘                                      │
│          │                                              │
│          ↓                                              │
│  ┌──────────────────────────────┐                      │
│  │ hasUnsavedChanges 计算属性   │                      │
│  └───────┬──────────────────────┘                      │
│          │                                              │
│          ↓                                              │
│  ┌─────────────────────────────────────────────┐       │
│  │  UnifiedSaveBar                             │       │
│  │  [状态]             [取消]  [保存]          │       │
│  └───────┬─────────────────┬──────────┬────────┘       │
│          │                 │          │                │
│          ↓                 ↓          ↓                │
│   启用/高亮           撤销      提交到配置               │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

**关键状态**：
1. **初始**：无修改，按钮禁用
2. **脏状态**：有修改，保存按钮蓝色，取消启用
3. **保存中**：保存按钮显示旋转器，禁用
4. **成功**："设置已保存"消息（3秒自动隐藏）
5. **错误**：显示错误消息，按钮保持启用

## 🔑 关键要点

1. **统一体验**：所有6个设置标签页（通用、供应商、路由、快捷键、行为、内存）都使用相同的保存/取消模式

2. **测试便利**：测试连接按钮从表单底部移到每个供应商卡片上，测试使用当前表单值（即使未保存）

3. **防止误操作**：
   - 修改不立即生效，需要点击"保存"
   - 点击"取消"可以撤销所有修改
   - 切换标签页/关闭窗口时有确认对话框

4. **清晰反馈**：
   - 保存按钮在有修改时显示蓝色高亮
   - 底栏显示"⚠️ 未保存的修改"状态消息
   - 测试结果直接显示在供应商卡片下方

5. **键盘友好**：Cmd+S保存，Escape取消

6. **无障碍支持**：完整的VoiceOver支持，符合WCAG AA标准

7. **性能优化**：
   - 脏状态检查高效（结构对比）
   - 保存操作异步执行，不阻塞UI
   - 支持并发测试多个供应商
