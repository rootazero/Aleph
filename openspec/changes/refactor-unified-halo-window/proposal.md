# Proposal: Refactor Unified Halo Window

## Summary

重构多轮对话Halo窗口，整合命令补全功能，新增类似Raycast的可伸缩副窗口（SubPanel），实现统一的交互入口。

## Motivation

当前存在两个独立的UI入口：
1. **命令补全窗口** (Cmd+Opt+/) - 用于斜杠命令补全
2. **多轮对话窗口** (Shift启动) - 用于持续对话

这种分离导致：
- 用户认知负担：需要记住两种不同的交互方式
- 热键冲突：不同模式使用不同热键
- 功能割裂：命令补全不能与对话无缝衔接
- UI不一致：两套窗口样式

## Proposed Changes

### 1. 统一入口热键
- **新热键**：`Cmd+Opt+/` 调出统一的Halo窗口
- **前置检测**：调出前检测光标是否聚焦于输入框
  - 如果未聚焦：弹出Toast提示"请先聚焦到输入框"
  - 如果已聚焦：显示Halo窗口，记录目标应用信息

### 2. 统一Halo窗口设计

```
┌─────────────────────────────────────────┐
│  Turn 1                      ESC 退出    │
│ ┌─────────────────────────────────────┐ │
│ │ 输入对话或命令...                    │ │  ← 主输入框
│ └─────────────────────────────────────┘ │
├─────────────────────────────────────────┤
│ ┌─────────────────────────────────────┐ │
│ │                                     │ │
│ │         副窗口 (SubPanel)           │ │  ← 动态伸缩
│ │                                     │ │
│ └─────────────────────────────────────┘ │
└─────────────────────────────────────────┘
```

### 3. 副窗口 (SubPanel) 功能

副窗口是一个多功能区域，根据上下文自动显示不同内容：

| 模式 | 触发条件 | 显示内容 |
|------|----------|----------|
| **命令补全** | 输入以 `/` 开头 | 命令列表（类似Raycast） |
| **AI选择器** | AI需要用户选择 | 选项列表（是/否、多选等） |
| **CLI输出** | AI执行后台操作 | 操作日志（滚动显示） |
| **确认对话** | 需要授权 | 确认按钮 + 描述 |
| **隐藏** | 无内容时 | 不显示副窗口 |

### 4. 副窗口设计规范

- **动态高度**：内容驱动，最小0（隐藏），最大300px
- **伸缩动画**：spring动画，0.3s
- **视觉风格**：
  - 圆角：8px
  - 阴影：0 4px 12px rgba(0,0,0,0.15)
  - 背景：半透明毛玻璃
  - 分隔线：1px subtle divider

### 5. 命令补全行为变更

- **不再上屏**：输入`/`时不会在目标应用输入字符
- **即时解析**：带着对话内容直接解析执行命令
- **示例流程**：
  1. 用户调出Halo，输入 `/en 你好世界`
  2. 副窗口显示命令补全列表（过滤显示`en`相关命令）
  3. 用户按Enter发送
  4. 路由解析 `/en` 命令，用"你好世界"作为输入
  5. AI翻译后，结果打印到目标应用

### 6. 光标聚焦检测

```
调用 Cmd+Opt+/ 热键
        ↓
检测当前应用是否有聚焦的文本输入区域
        ↓
    ┌───┴───┐
  有聚焦   无聚焦
    ↓        ↓
记录目标   Toast提示
应用信息   "请先点击输入框"
    ↓
显示Halo窗口
```

### 7. 删除旧组件

- 删除独立的 CommandModeCoordinator
- 删除 commandMode 状态
- 统一到新的 UnifiedHaloWindow

## Impact Analysis

### 修改文件
- `HaloWindow.swift` - 重构为统一窗口
- `HaloView.swift` - 新增SubPanel组件
- `HaloState.swift` - 新增SubPanelMode枚举
- `CommandModeCoordinator.swift` - 删除
- `ConversationInputView.swift` - 整合到统一窗口
- `CommandListView.swift` - 适配SubPanel
- `AppDelegate.swift` - 更新热键处理
- `EventHandler.swift` - 更新事件处理

### 新增文件
- `SubPanelView.swift` - 副窗口组件
- `SubPanelState.swift` - 副窗口状态管理
- `FocusDetector.swift` - 光标聚焦检测工具
- `UnifiedInputCoordinator.swift` - 统一输入协调器

### 删除文件
- `CommandModeCoordinator.swift` - 功能合并到统一协调器

## Risks & Mitigations

| 风险 | 缓解措施 |
|------|----------|
| 光标聚焦检测可能不准确 | 使用Accessibility API + fallback逻辑 |
| 副窗口动画可能卡顿 | 使用native animation，限制重绘频率 |
| 热键冲突 | 保留旧热键作为别名（过渡期） |
| 用户习惯改变 | 提供迁移提示，渐进式过渡 |

## Out of Scope

- MCP工具集成（单独提案）
- Skills系统（单独提案）
- 多语言输入法优化（单独提案）

## Success Criteria

1. 单一热键 `Cmd+Opt+/` 可调出统一窗口
2. 命令补全在副窗口中正确显示
3. 副窗口高度动态伸缩流畅
4. 光标未聚焦时正确提示
5. AI输出正确打印到目标应用
6. 所有现有测试通过
